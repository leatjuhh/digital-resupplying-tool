# Sessie 18 April 2026 - Store-exclusielijst, UI-polish & DB-bugfix

## Doel van de sessie

Drie onafhankelijke verbeteringen:
1. Outlet- en administratieve filialen uitsluiten van herverdeling via een centrale configuratie
2. Proposal detail en loginpagina visueel verrijken
3. Kritieke DB-bugfix: proposal-approval blokkeerde met een 500-fout

---

## Afgeronde onderdelen

### 1. Store-exclusielijst (`backend/redistribution/store_config.py`)

Nieuw bestand met:
- `NON_REDISTRIBUTION_STORES`: frozenset met filiaalcodes voor outlet (14, 15, 16, 39), administratief (0, 2, 3, 27, 99) en gesloten (35)
- `is_redistribution_candidate(store_code: str) -> bool` als enkelvoudige toegangspoort

`algorithm.py` bijgewerkt op twee plekken:
- `calculate_batch_store_totals`: filtert niet-kandidaten uit de GROUP BY-resultaten
- `load_article_data`: slaat records van uitgesloten filialen over

### 2. Proposal detail UI (`frontend/components/proposals/proposal-detail.tsx`)

Hulpfuncties toegevoegd:
- `parseHoofdgroep()` — strips `Artikelgroep : X` suffix
- `parseSeizoenjaarInfo()` — splitst jaar, collectie en bestelcode uit het Seizoenjaar-veld
- `getColorSwatch()` — vertaalt Nederlandse/Engelse kleurnaam naar hex
- `MetaField` component — consistent label+waarde blokje

Bestelcode-resolutie: dedicated veld → parsed seizoenjaar → artikelnummer als fallback.

Tabs in header (`frontend/app/proposals/[id]/page.tsx`):
- Voorstel / Analyse tabs naast de actieknoppen

### 3. Loginpagina & achtergrond

`login-page-client.tsx`:
- `LiveStat` component met flash-animatie bij update
- Toont live telling van verdelingen, filialen en trends

`network-background.tsx`:
- Algoritme-node + winkelknopen vervangt magazijn + artikel-nodes
- Oranje herverdelingsflows (surplus → shortage beams)
- Surplus/shortage-ringindicatoren per winkelknoop

### 4. DB-bugfix: `feedback.rating` NOT NULL

**Probleem:** `POST /api/pdf/proposals/{id}/approve` gaf 500:
```
sqlite3.IntegrityError: NOT NULL constraint failed: feedback.rating
```

**Oorzaak:** Database aangemaakt toen `rating` en `comment` nog `NOT NULL` waren; SQLAlchemy-model later versoepeld naar `nullable=True` zonder bijbehorende migratie.

**Fix:** Eenmalig migratiescript uitgevoerd — `feedback` tabel gerecreëerd met nullable kolommen, bestaande data behouden. Script daarna verwijderd.
